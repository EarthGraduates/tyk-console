## 二、执行内容

### 2.1 迁移文件

`supabase/migrations/001_init.sql` — 4 张表的完整 DDL + COMMENT + 索引 + 预置数据。

### 2.2 数据库服务层

`src/providers/ichse-db.ts` — 独立 Supabase 客户端（schema=`ichse`），提供：

| 模块 | 方法 |
|------|------|
| `apiDefinitionsDb` | list / getByApiId / create / update / delete / markSynced / markSyncFailed |
| `apiDefinitionLogDb` | insert / listByApiId |
| `apiKeysDb` | list / getByKeyId / create / revoke |
| `usersDb` | ensureUser / getSystemUserId |

### 2.3 DataProvider 双写改造

`src/providers/tyk-data-provider.ts` — 5 个辅助函数 + 6 处调用点：

| 操作 | 原有流程 | 新增 DB 操作 |
|------|---------|-------------|
| create apis | POST /tyk/apis/ → reload | INSERT api_definitions + INSERT log |
| update apis | PUT /tyk/apis/{id} → reload | UPDATE api_definitions + INSERT log |
| delete apis | DELETE /tyk/apis/{id} → reload | UPDATE status='inactive' + INSERT log |
| create keys | POST /tyk/keys/create → reload | INSERT api_keys |
| delete keys | DELETE /tyk/keys/{id} → reload | UPDATE status='revoked' |

**容错策略**：DB 操作失败仅 console.warn，不影响 Tyk 主流程。
