## 二、修复的 Bug

| Bug | 根因 | 修复 |
|-----|------|------|
| 匿名 PostgREST 请求返回"用户不存在" | `db_pre_request` 无 JWT 时不抛异常，`v_user_id` 为 NULL 继续查 users 表 | 添加 `IF v_user_id IS NULL` 显式检查 |
| JWT payload 解码失败 | `atob` 不支持 base64url，payload 含中文 UTF-8 多字节 | base64url→base64 转换 + Uint8Array.TextDecoder |
| PostgREST TEXT 返回值损坏 JWT | PostgREST 序列化 TEXT 插入转义序列 | login() 返回 JSONB，前端 JSON.parse 直取 |
| pgjwt 签名包含空格 | `translate(E'+/=\n', '-_  ')` 把 `=`、`\n` 映射为空格 | 改为 `translate(E'+/=\n', '-_')`，多余字符自动删除 |
| db_pre_request 被 RLS 阻止 | 非 SECURITY DEFINER，无法查 ichse.users | 添加 SECURITY DEFINER |
| db_pre_request 写操作被拒 | PostgREST 只读事务不支持 UPDATE rate_limit | 移除速率限制 |
| 未登录直接进仪表板 | 缺少 `<Authenticated>` 包裹受保护路由 | 添加 Authenticated + Navigate to /login |

## 四、启动方式

```bash
# Tyk Gateway
docker-compose -f ~/Hermes/git-tyk/tyk-gateway-docker/docker-compose.yml up -d

# PG + PostgREST
docker-compose -f docker-compose.yml down && docker-compose -f docker-compose.yml up -d
cat database/postgresql/schema.sql | docker-compose exec -T postgres psql -U ichse -d ichse
cat database/postgresql/migrations/002_auth_and_rls.sql | docker-compose exec -T postgres psql -U ichse -d ichse
cat database/postgresql/migrations/003_functions.sql | docker-compose exec -T postgres psql -U ichse -d ichse
cat database/postgresql/migrations/004_rls.sql | docker-compose exec -T postgres psql -U ichse -d ichse
docker-compose restart postgrest

# 前端
npm run dev
```
