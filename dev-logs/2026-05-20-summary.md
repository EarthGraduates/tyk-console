## 一、Bug 修复（3 个）

### Bug #1：服务页/密钥页表格操作列溢出
- **根因**：Table 缺少 `scroll` 属性，列宽挤压导致按钮溢出
- **修复**：`scroll={{ x: 'max-content' }}`
- **提交**：`40eb942`

### Bug #2：服务详情路由跳转改为 Drawer 弹出
- **问题**：点击详情跳转后页面空白
- **修复**：用 Drawer 替代路由跳转，`useOne` 查询数据
- **踩坑**：`useOne` 返回 `{ query, result }`，不是 `{ data, isLoading }`（与 `useList` 不同）
- **提交**：`f4ff899`、`a6b54b8`（暗色代码主题）

### Bug #3：仪表板 health 数据恒 0
- **根因 1**：Tyk `enable_health_checks: false`
- **根因 2**：代码读 `health.requests`，Tyk 返回 `average_requests_per_second`
- **修复**：启用 health check + 字段映射 + 清除误导列
- **提交**：`ef46afb`、`7f1b03e`

## 七、经验教训

1. **查 bug 先看数据源**：仪表板 health 恒 0 → 不是代码问题，是 Tyk 配置和字段映射两层问题
2. **localStorage 残留是常见坑**：旧 key 值污染新配置，换 key 名是最干净的隔离方案
3. **useOne ≠ useList**：Refine v5 中 `useOne` 返回 `{ query, result }`，和 `useList` 的 `{ data, isLoading, result }` 不同
4. **规范先行**：Table 必须配搜索/分页/排序，不是「数据少就先不加」
5. **Tyk health endpoint 默认关闭**：OSS 版本需要手动 `enable_health_checks: true`
