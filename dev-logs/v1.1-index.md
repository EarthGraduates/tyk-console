## 二、架构概览

```
浏览器 (localhost:5173)
  ├── /tyk/* → Tyk Gateway (localhost:8080)        ← API/密钥 CRUD
  ├── /hello → Tyk Gateway                          ← 健康检查
  └── /db/*  → PostgREST (localhost:3001)           ← 管理数据库
                    └── PostgreSQL (localhost:5433)

  ├── tykFetch() → Tyk REST API (主流程)
  └── ichse-db  → fetch(/db/...)   (双写，best-effort)
```

## 五、数据库设计（Schema v1.0）

### 5.1 表清单

| 表 | 字段数 | 用途 |
|----|:---:|------|
| `ichse.users` | 7 | 业务用户（预置 system / public） |
| `ichse.api_definitions` | 19 | API 完整配置 + 自管状态 |
| `ichse.api_definition_log` | 8 | 变更日志 |
| `ichse.api_keys` | 14 | 密钥管理 |

### 5.2 核心字段语义

| 字段 | 表 | 语义 |
|------|----|------|
| `status` | api_definitions | `active`(已推送) / `inactive`(DB保留,Tyk已删) / `archived` |
| `sync_status` | api_definitions | `synced` / `pending` / `failed` |
| `owner_id` | api_definitions | FK→users，权限归属 |
| `version` | api_definitions | 递增版本号，配合 log 表做回滚 |
| `key_value` | api_keys | 开发阶段明文，生产需加密 |
