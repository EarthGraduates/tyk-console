## 二、架构概览

```
浏览器 (localhost:5173)
  ├── /tyk/* → Tyk Gateway (localhost:8080)        ← API/密钥 CRUD（tyk-data-provider）
  ├── /hello → Tyk Gateway                          ← 健康检查
  └── /db/*  → PostgREST (localhost:3001)           ← 管理数据库 + JWT 认证
                    └── PostgreSQL (localhost:5433)
                         ├── ichse schema（业务表）
                         └── pgjwt / pgcrypto（加密函数）

JWT 认证流程：
  POST /db/rpc/login → PG login() → 校验密码 → pgjwt.sign() → {token}
  → 前端存 localStorage("ichse_jwt")
  → 后续请求 ichse-db data provider 自动加 Authorization: Bearer header
  → PostgREST db_pre_request 解析 JWT + 用户状态检查 → 设 app.role 供 RLS

Refine DataProvider
  ├── default → ichse-db（PostgREST，带 JWT 认证）
  └── tyk    → Tyk Gateway（API CRUD / 密钥 CRUD）
```

> **注意**：Tyk 的 docker-compose 有本地修改——`virtiofs` 不支持单文件 bind mount，改为目录挂载 + `command: --conf=` + `user: root`。详见 `deploy/deployment.md` §6.1。

启动命令（2026-05-22 更新）：
```bash
# Tyk Gateway
docker-compose -f ~/Hermes/git-tyk/tyk-gateway-docker/docker-compose.yml up -d

# PG + PostgREST（重建加载新 JWT 配置）
docker-compose -f docker-compose.yml down && docker-compose -f docker-compose.yml up -d

# 首次建表 + 迁移（用管道注入！heredoc 可能不生效）
cat database/postgresql/schema.sql | docker-compose exec -T postgres psql -U ichse -d ichse
cat database/postgresql/migrations/002_auth_and_rls.sql | docker-compose exec -T postgres psql -U ichse -d ichse
cat database/postgresql/migrations/003_functions.sql | docker-compose exec -T postgres psql -U ichse -d ichse
cat database/postgresql/migrations/004_rls.sql | docker-compose exec -T postgres psql -U ichse -d ichse
docker-compose restart postgrest

# ===== 前端 =====
npm run dev
```

## 五、数据库设计（Schema v2.0）

### 5.1 表清单

| 表 | 字段数 | 用途 |
|----|:---:|------|
| `ichse.users` | 14 | 业务用户 + 认证（password_hash / status / secret_level / 三员+viewer+business_user） |
| `ichse.api_definitions` | 20 | API 完整配置 + 自管状态 + secret_level |
| `ichse.api_definition_log` | 7 | 变更日志 |
| `ichse.api_keys` | 15 | 密钥管理 + secret_level |
| `ichse.audit_log` | 15 | 安全审计日志（append-only，HMAC 防篡改） |
| `ichse.rate_limit` | 2 | 速率限制（已创建未启用，待 Phase 2） |

### 5.2 核心字段语义

| 字段 | 表 | 语义 |
|------|----|------|
| `status` | api_definitions | `active`(已推送) / `inactive`(DB保留,Tyk已删) / `archived` |
| `sync_status` | api_definitions | `synced` / `pending` / `failed` |
| `owner_id` | api_definitions | FK→users，权限归属 |
| `version` | api_definitions | 递增版本号，配合 log 表做回滚 |
| `key_value` | api_keys | 开发阶段明文，生产需加密 |
