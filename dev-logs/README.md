# Dev Logs

开发日志目录。**只保留阶段总结和版本总结**，不放单日碎片或单 bug 修复记录。

## 命名规则

- 阶段总结：`YYYY-MM-DD-slug.md`
- 版本总结：`vX.Y-summary.md`

## 什么值得记

- 一个 phase/stage 完成后的总结（做了什么、关键决策、遗留问题）
- 版本发布总结（变更清单、测试结果、已知问题）
- 架构/方案讨论的结论（讨论什么、结论是什么、为什么）

## 什么不值得记

- 单日工作碎片（"今天修了 3 个 bug"）
- 单个 bug 的修复过程（这些在 git commit message 里）
- 单文件/组件的小改动说明
- 临时调查笔记

## 如何记

每个文件的格式：

```markdown
# {标题}

> 日期：YYYY-MM-DD | 涉及：{模块/组件}

## 做了什么

- 要点 1
- 要点 2

## 关键决策

- 决策 + 原因

## 遗留问题

- [ ] 待办 1
```
