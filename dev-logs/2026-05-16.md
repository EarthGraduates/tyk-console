## 一、Bug 修复阶段

### 1.1 服务页 API 列表 No data

- **根因**：Refine v5 `useList` 需 resource 注册 + 用 `result` 属性读取
- **修复**：App.tsx 注册 `apis`/`keys` 资源 → `useList` 改用 `result?.data`

### 1.2 仪表板状态与服务页不一致

- **根因**：仪表板只看 `/tyk/health/` 的 error 数，不看 `active: false`
- **修复**：状态列先检查 active → 停用显示「已停用」标签

### 1.3 antd v5 message 被静默吞掉

- **根因**：静态 `message.success/error` 在 antd v5 不生效
- **修复**：改用 `App.useApp()` 获取 message 实例

### 1.4 首次访问 403 — API Secret 未配置

- **根因**：Secret 默认值为空，用户不留神没填
- **修复**：Data Provider `getSecret()` 默认返回 `'foo'`（dev 环境内置值）

### 1.5 创建 API 弹窗改为 Modal

- **要求**：点击「创建 API」不跳独立页，弹窗完成配置
- **实现**：6 Tab Modal → 提交成功后自动关闭

### 1.6 密钥页模块修复

- **创建密钥无反应**：`Modal.info` 静态方法不稳定 → 改为受控 Modal
- **创建失败 400**：Tyk 强制要求 `access_rights` → 新增授权 API 下拉选择器
- **列表 404**：Tyk OSS 禁止列出密钥 → 开启 `enable_hashed_keys_listing`
- **列表 No data**：列出的是哈希字符串 → Data Provider 逐条查详情并注入 key_id
- **列表字段缺失**：Tyk 详情不含 `key_id` → Data Provider 注入
- **编辑崩溃**：`useUpdate` 无参调用 + `DatePicker` 在 Refine v5 环境不稳定
  - 状态：暂时回滚到 ab55a92 版本，待后续修复
