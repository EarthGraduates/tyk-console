# ADR-0001: 停用 API 不处理关联密钥

V1.1 实现 `status='inactive'`（停用）流程时，明确决定：停用操作仅从 Tyk DELETE API 定义，不处理该 API 关联的密钥。密钥在 Tyk 中保持原样，成为孤儿引用（指向已删除的 API ID），不会自动吊销或清理。

**理由**：Tyk OSS 的 keys 和 APIs 是独立对象——DELETE API 本身也不会清理 keys。密钥生命周期管理（吊销、重新绑定到新 API ID）是一个独立功能，需要独立设计。V1.1 的目标是 stabilization，在未充分设计的情况下自动处理密钥会引入不必要的行为假设。

**后果**：重新启用一个停用的 API 时，旧密钥不会自动恢复。重新启用后用户需要手动重新创建或重绑密钥。这个限制应在 UI 上提示。

**替代方案**：
- 停用时自动吊销关联密钥 → 被拒绝，因为增加了 V1.1 的范围，且「重新启用后密钥是否恢复」缺乏明确的行为定义
- 停用时保留密钥但标记为 suspended → 被拒绝，因为 Tyk OSS 没有密钥挂起状态
